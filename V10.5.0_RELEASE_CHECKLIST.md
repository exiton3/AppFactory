### Common Questions Anticipated

**Q: Should I migrate from the legacy AppFactory.Framework.Messaging package?**  
A: The legacy package still works. New projects should use the new multi-cloud packages. Existing projects can migrate gradually.

**Q: Which messaging package should I use - AWS or Azure?**  
A: Use Messaging.Aws for AWS SQS/Lambda, Messaging.Azure for Azure Service Bus/Queue Storage/Functions. The Core package provides abstractions for both.

**Q: Can I switch from AWS to Azure later?**  
A: Yes! That's the whole point. Your business logic (handlers, publishers) stays the same. Only DI registration changes.

**Q: What about other platforms (GCP, RabbitMQ)?**  
A: Planned for future releases. The Core abstractions are ready - just need platform-specific implementations.

**Q: Where are the unit tests?**  
A: Core package has 22 tests. AWS/Azure tests will be added in v10.5.1 (non-blocking).
